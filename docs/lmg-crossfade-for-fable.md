# LMG — кроссфейд (стриминг, форк Media3) для Fable

Clean-room спека по механике, разобранной из реализации Apple Music. **Код Apple НЕ портим** —
описан алгоритм/математика (это стандартный аудио-инжиниринг), Fable пишет **свой** класс. ExoPlayer/Media3
под Apache-2.0 → форкать и модифицировать можно свободно; добавления Apple — их копирайт, не копируем.

Область: **стриминговый путь (Media3)**. Офлайн (Juce/Oboe) делает то же самое родным микшером (те же кривые) —
отдельно, здесь не трогаем.

## Почему без форка нельзя (и почему НЕ два плеера)

Стоковый ExoPlayer/Media3 держит **один** аудио-рендерер и не умеет играть два media-периода параллельно.
Кроссфейд требует, чтобы уходящий (lead) и входящий (follow) треки звучали **одновременно на двух аудио-
рендерерах в ОДНОМ пайплайне** (один клок, сэмпл-точное выравнивание). Два отдельных плеера = два клока =
дрейф/артефакты (для стрима особенно). Поэтому — форк Media3 с двумя рендерерами.

## Архитектура форка (как у Apple, но свой код)

1. **`FadeControl`** — новый компонент плеера, передаётся в форкнутый `ExoPlayer.Builder` отдельным параметром
   (наравне с LoadControl/TrackSelector).
2. **Два аудио-рендерера**: `createRenderers()` создаёт 2 аудио-рендерера; `MediaPeriodQueue`/`ExoPlayerImplInternal`
   форкаются так, чтобы держать **два периода enabled одновременно**; `MediaPeriodHolder` получает `getRendererIdx()`.
3. **Громкость на рендерер**: `renderer.handleMessage(MSG_SET_VOLUME, Float)` (в Media3 `Renderer.MSG_SET_VOLUME`).
4. **Хуки из playback-loop** на границе периодов: `canFade(lead,follow)` → `prepare(lead,follow)` (разово) →
   `maybeStart(...)` → `doCrossFade(...)` (каждый tick).

## Тайминг (ключевое — сэмпл-точность)

Фейд гонится по **аудио-часам уходящего рендерера**: `renderers[leadIdx].mediaClock.positionUs`. НЕ по
wall-clock. `System.currentTimeMillis()` — только для троттлинга сообщений громкости.

## Кривые громкости (математика — стандартный DSP, использовать можно)

Уровень считается по нормализованному `t∈[0,1]`. `c` — коэффициент кривизны (дефолт ~10) для EXP/LOG/SIGMOID.

**fade-IN (растёт 0→1):**
- LINEAR: `t`
- CUBIC: `t^3`
- EXPONENTIAL: `(c^t − 1)/(c − 1)`
- LOGARITHMIC: `log(c·(1−t) + t)/log(c)` (Apple-вариант — «быстрый старт»)
- **CONSTANT_POWER: `sqrt(t)`** (equal-power)
- SIGMOID: `1/(1 + e^{−(t−0.5)·c})`

**fade-OUT (падает 1→0):** зеркально — `1−in(1−t)`; в частности CONSTANT_POWER: `sqrt(1−t)`.

**Дефолт:** Apple ставит **fade-in = LOGARITHMIC, fade-out = EXPONENTIAL**. Для кроссфейда я бы рекомендовал
**CONSTANT_POWER на обе стороны** (equal-power) — держит постоянную суммарную мощность в зоне перекрытия, нет
провала громкости на середине. Дать выбор кривой в настройках (опционально), дефолт equal-power.

## MANUAL режим (это и делаем — достижимо и качественно)

Симметричный фейд фикс-длительности `D` (слайдер 1..12 c):
- **fade-out** уходящего начинается за `D` до конца трека: `x = ((trackEnd) − D − posInLead)/D`, `x:1→0` → кривая.
- **fade-in** входящего: период **пре-роллится на `D`** (стартует за `D` до конца lead), `t` идёт от 0 к 1 по
  мере входа → кривая растёт 0→1. Оба на своих рендерерах, громкости встречно.
- Завершение: `fadeOutLevel < 0.05 && fadeInLevel > 0.95`, либо EOS уходящего.

## Троттлинг сообщений громкости

Не чаще шага: `step = D/20`, зажатый в `[200мс, 500мс]` (≈20 шагов на фейд). В `doCrossFade` — если
`now − lastMsg < step` → пропускаем tick. Достаточно плавно, не спамит рендерер.

## Гейтинг (когда кроссфейд НЕ применять)

- **Соседние треки одного альбома → gapless, НЕ кроссфейд.** Критично: альбом слушают слитно (gapless), а
  кроссфейд для переходов между несвязанными треками (плейлист/шаффл/радио). Сравнивать по album id + соседству.
- repeat-one — выключить.
- Трек короче `2·D` — фейдить нечего, пропустить.
- Слабые устройства (по желанию — фиче-гейт), чтобы не тянуть два рендерера.
- Пауза во время перехода: физически `stop()`/`start()` уходящего рендерера (Apple так и делает), чтобы фейд
  замирал корректно.

## Настройки (UI)

Три взаимоисключающих варианта = состояние: **Off / Manual / Automatic**. Слайдер длительности **1..12 c**
активен только в Manual. В LMG-glass-стиле (сегмент-контрол + слайдер).
- **Off / Manual — делаем сейчас.**
- **Automatic** — у Apple это «seamless»: нативный C++ `LevelComposer`/`CrossfadeRecipe` строит музыкально-осмысленные
  кривые по `AudioAnalysis` трека (биты/лоудность/точки фейдов), подбирает точку пересечения. Это **огромный
  отдельный кусок** (нужен per-track аудио-анализ + нативный композер). **Пропускаем на старте**, можно вернуться
  позже упрощённой эвристикой (например, точка перехода по затуханию RMS в конце трека). Для 95% ценности хватает
  Manual.

## Definition of Done (стрим)

- Форк Media3 с двумя аудио-рендерерами; `FadeControl` в билдере; свой код (не порт Apple).
- Manual-кроссфейд 1..12 c, equal-power по умолчанию, драйв по аудио-часам lead-рендерера, троттлинг ~20 шагов.
- Гейтинг: соседи-по-альбому → gapless; repeat-one/короткий трек/пауза — корректно.
- Настройки Off/Manual(+слайдер); Automatic скрыт/заглушён до отдельной итерации.
- Сборка зелёная, на переходе плейлист-треков слышен ровный кроссфейд без провала громкости и щелчков.

## Правовая рамка (коротко)

- Форк **Media3 (Apache-2.0)** — свободно.
- **Математика кривых и техника «два рендерера + рампы»** — общий аудио-инжиниринг, использовать можно.
- **Код Apple (их класс/интерфейс/структуру) не копируем** — реализуем то же поведение своим кодом.
- AUTOMATIC-композер Apple (нативный seamlesscomposer) — их проприетарщина, не тащим; свою эвристику — если захотим.
