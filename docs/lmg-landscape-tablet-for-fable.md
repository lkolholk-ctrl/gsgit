# LMG — адаптация под альбомную/планшет (для Fable)

Цель: приложение полностью юзабельно в **ландшафте** (HDMI/ПК-режим, планшет). Эталон раскладки —
трёхколоночная схема (как в референсе AB Player), но в **liquid-glass стиле LMG**, НЕ плоский вид.
`FullPlayer` уже адаптирован — остальные экраны подтягиваем к общему скелету.

## Принцип (не дублировать экраны!)

Один и тот же экран **перекладывает** контент по размеру окна, а не две копии:
```kotlin
val widthClass = calculateWindowSizeClass(activity).widthSizeClass
val expanded = widthClass == WindowWidthSizeClass.Expanded   // ландшафт/планшет
// Medium (≈планшет-портрет / крупный телефон-ландшафт) — по вкусу: чаще как expanded
```
- **Compact** (телефон-портрет) → текущая раскладка (вертикальный стек + bottom-bar + мини-плеер).
- **Expanded** (ландшафт/планшет) → новый скелет ниже.
- Разбивку делать **внутри** экрана (общий VM/стейт), а не форкать навигацию.

## Скелет ландшафта (общий каркас приложения)

```
┌──────┬─────────────────────────┬───────────────────┐
│ Rail │  Основной контент       │  Контекст-сайдбар │
│ (nav)│  (скроллится)           │  Now Playing +    │
│      │                         │  Queue (Up Next)  │
├──────┴─────────────────────────┴───────────────────┤
│  Плеер-бар на всю ширину (постоянный, glass)        │
└─────────────────────────────────────────────────────┘
```
- **NavigationRail слева** (glass): Home / Video / Explore / Favorites / Library / Settings. Заменяет
  нижний бар в ландшафте. Иконки + подписи, активный — акцентом темы.
- **Центр** — контент активного раздела, скроллится независимо.
- **Правый сайдбар** (glass, ширина ~300-360dp) — **Now Playing + Queue (Up Next)**. Виден на Home и
  большинстве экранов; можно прятать по кнопке, если контенту нужна ширина.
- **Плеер-бар снизу на всю ширину** — постоянный, не перекрывает контент (контент имеет `paddingBottom`).
  Мини-плеер портрета в ландшафте становится этим баром + правым сайдбаром.
- Каркас — один общий `AdaptiveScaffold(rail, content, sidebar, playerBar)`; экраны отдают только `content`.

## По экранам

- **Home** — центр: Recently Played (горизонтальный ряд) + All Songs (список). Справа: Now Playing + Queue.
- **Video (клипы)** — НОВЫЙ раздел (ICM добавили клипы). Сетка превью (2-4 колонки по ширине), тап →
  видео-плеер (ExoPlayer video surface). В ландшафте видео крупно слева, список/похожие справа.
  *Эндпоинты ICM для клипов уточняются — данные пойдут через `/lmg/icm/*` (ключ/origin/UA ставит сервер).*
- **Explore** — сетки/подборки в несколько колонок (в ландшафте больше колонок, `LazyVerticalGrid` с
  адаптивным `GridCells.Adaptive(minSize)`).
- **Search** — слева поле+результаты, справа — превью/очередь. Или результаты в грид на всю ширину.
- **Favorites / Library** — список слева, деталь (плейлист/альбом → треклист) справа: **мастер-деталь**.
- **Album / Playlist** — арт+инфо слева, треклист справа (в портрете — стек).
- **Settings** — разделы слева (в рамках центра), панель настроек справа (мастер-деталь).
- **FullPlayer** — уже готов (обложка слева, управление справа) — свериться, что вписан в общий каркас.

## Стиль (важно — не копировать AB Player визуально)

- Рейл, сайдбар, плеер-бар — **на стекле LMG** (drawBackdrop/blur/vibrancy), акцент из темы, а не плоский фон.
- Радиусы, тени, highlight — как в остальном приложении. Берём **раскладку**, не внешний вид референса.

## Технические требования

- **`material3-window-size-class`** для брейкпоинтов (не хардкодить orientation).
- **Стейт переживает поворот**: экранный стейт в `ViewModel` (или `rememberSaveable`), чтобы список/скролл/
  выбор не сбрасывались при смене ориентации. (Тот же урок, что с device-flow: `remember` не переживает
  recreation.)
- **`NavigationRail`** из material3 в expanded, `NavigationBar` в compact — одна навигация, две подачи.
- Реюз: секции (RecentlyRow, SongRow, QueueList, NowPlayingCard, ClipTile) — общие Composable, оба режима
  их переиспользуют.
- Не ломать портрет: compact-ветка = текущее поведение 1:1.

## Definition of Done

- В ландшафте: рейл слева, контент в центре, Now Playing+Queue справа, плеер-бар снизу на всю ширину.
- Все экраны (Home, Video, Explore, Search, Favorites, Library, Album/Playlist, Settings, FullPlayer)
  используемы в ландшафте, без растянутого портрета и обрезаний.
- Поворот не сбрасывает стейт. Портрет работает как раньше. Всё в glass-стиле LMG. Сборка зелёная.

## Клипы (video) — отдельный под-пункт, нужны эндпоинты

Чтобы раздел Video заработал end-to-end, пришли форму клип-ручек ICM (путь/ответ, напр. `/clips`,
`/clip/{id}`, поля `video_url`/`hls`/качество/обложка). Тогда:
- сервер: подтверждаю, что брокер проксирует их без правок (ключ/origin/UA уже ставятся), при нужде добавлю
  latency-группу `video`;
- клиент: опишу data-слой (IcmApi добавляет clip-методы) + ExoPlayer video (тот же плеер, что аудио, умеет
  видео из коробки).
