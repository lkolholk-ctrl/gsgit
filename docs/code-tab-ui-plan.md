## Где мы сейчас (после Стадий 0-6)
- **Browser** (`CodeBrowser.kt`, 238): крошки `branch / seg / seg` → recent-чипы → hairline →
  строка `N unsaved · commit · discard` → список (иконка + имя + размер + точка-грязный).
- **Changes panel** (`CodeChangesPanel.kt`, 120): `← changes (N) · commit` → плоский список
  `M  имя / dir  ✕`.
- **Editor**: переиспользуемый `CodeEditorScreen` (1778 строк, тяжёлый chrome: mode-pills,
  accessory-bar, и т.д.) — открывается full-screen.
- **Commit sheet** (`CodeCommitSheet.kt`): простой `AiModuleAlertDialog` с полем сообщения.
- Эстетика — моноширинный терминал (JetBrains Mono), плоские ряды.

## Что слабовато (честная оценка)
1. **Плоская иерархия.** Имя файла, размер, путь, маркеры — почти одного веса; глаз не цепляется.
2. **Управляющие строки — текст вплотную.** `N unsaved · commit · discard` и `← changes` —
   кликабельный текст без явных аффордансов (не выглядит как кнопки/бар).
3. **Иконки типов бедные и одноцветные** (5 вариантов, все `textSecondary`). VS Code/файл-менеджеры
   цветят по типу — это читается мгновенно.
4. **Состояния loading/empty — голый текст** («loading…», «empty», «no changes»). По полиш-принципам
   нужны skeleton/иконка+подпись.
5. **Нет переходов** browser ↔ editor ↔ changes — резкие подмены, теряется ощущение «одного места».
6. **Editor — чужой тяжёлый chrome**, не «мини-IDE»: дублирует репо-навигацию, много кнопок не в тему
   Code-контекста.
7. **«M»-маркер и грязные точки невзрачные** — один цвет, мелкие; тип изменения (M/A/D) не различается.

## Оси улучшения (приоритезировано)

### P1 — Source Control: бар изменений + панель (самое заметное)
- **Sticky Source-Control bar** вместо текстовой строки `N unsaved`: компактный бар с иконкой ветки,
  числом изменений, и явными кнопками **commit** (filled-pill, accent) / **discard** (outline).
  Появляется только при наличии черновика.
- **Цветовая кодировка типов изменений** в панели и на рядах: `M` (modified) — warning/amber,
  будущие `A` (added) — green, `D` (deleted) — error. Сейчас всё `M`/точка одного цвета.
- **Панель изменений как bottom-sheet** (выезжает снизу над браузером), а не полная подмена экрана —
  тогда видно и дерево, и изменения (как VS Code SCM-вид). Альтернатива: оставить отдельной
  поверхностью, но с заголовком-баром и группировкой по папкам.
- **Commit sheet → нормальный bottom-sheet**: число файлов, список изменённых (свернуто), поле
  сообщения с подсказкой, primary-кнопка. Сейчас — alert-диалог.

### P2 — Browser polish (ряды, крошки, иконки, состояния)
- **Ряд файла:** имя — `textPrimary` Medium; путь/размер — мельче, `textMuted`; директории — Bold +
  chevron-вправо как аффорданс захода. Чёткий ритм отступов (8/12/16).
- **Цветные иконки по типу** (kt/js/py/md/img/json…) — палитра из темы (syntax-токены уже есть:
  `syntaxKeyword/String/Number…`), не хардкод.
- **Крошки:** текущий сегмент — Bold accent, родители — кликабельные `accent` с разделителем-chevron;
  доб. иконку ветки слева.
- **Состояния:** loading → skeleton-ряды (3-4 серых плейсхолдера); empty → иконка папки + «empty
  directory»; error → иконка + «failed to load» + **retry**-кнопка (сейчас просто текст без ретрая).
- **Recent-чипы:** скруглённые, с иконкой типа + усечением; «recent» как тихий лейбл.
- Опц.: строка-`..` (на уровень вверх) для тех, кто не пользуется крошками.

### P3 — Editor chrome + переходы (крупнее, частично отложенный Phase 2)
- **Переходы:** browser → editor — slide/fade; changes panel — slide снизу. Без агрессии (по полиш-оси
  «scale 0.98 на press, fade на появление»).
- **Лёгкий Code-editor chrome:** сейчас открывается полноразмерный `CodeEditorScreen` со всем его
  UI. Для Code-контекста хочется тоньше: путь-крошки сверху, индикатор «draft/dirty», кнопки
  save-to-draft / back, без лишних режимов. **Это и есть отложенный Phase-2 редактора** (сделать его
  «тупым»: content+path+language+saveCallback) — крупная работа, делать отдельным заходом.

## Принципы (CLAUDE.md)
