## 0. Что ты строишь

Единая **панель администратора** для управления серверной частью приложения GsGit
(GitHub-клиент под Android). Это НЕ пользовательское приложение — это личный пульт
владельца. Оно ходит на HTTP API сервера `api.gsgit.org` и позволяет:

- включать/выключать техработы (kill-switch всех клиентов);
- задавать версии и changelog для гейтов обновления;
- рассылать пуш-анонсы всем устройствам;
- смотреть список зарегистрированных устройств и сводную статистику.

Приложение приватное, распространяться не будет. Один пользователь — владелец.

## 1. Платформа и стек (рекомендация)

Почему: проще всего, ты это делаешь лучше всего, не нужен Android-билд/подпись/CI,

но веб проще и быстрее. Сделай веб, если явно не сказано иное.

## 2. Авторизация (важно — читай внимательно)

Все админ-запросы авторизуются заголовком:

```
X-Admin-Key: <ключ>
```

- Ключ вводится один раз на экране входа, хранится **локально** (localStorage для веба /
  EncryptedSharedPreferences для Android). Никогда не хардкодь ключ в коде.
- Ключ — это правда сервера. Клиент его не проверяет сам; проверку делает сервер и
  возвращает `401 {"error":"bad admin key"}` при неверном.
- **Флоу входа:** пользователь вводит ключ → приложение делает пробный запрос
  `GET /admin/stats` с этим ключом → если `200`, ключ верный, сохраняем и пускаем в
  панель; если `401`, показываем «Wrong key», ключ не сохраняем. **Поля управления
  показывай ТОЛЬКО после успешной проверки**, не раньше (не раскрывай интерфейс по
  первому символу).
- Кнопка «Lock» / «Logout» стирает ключ и возвращает на экран входа.

## 3. API-контракт бэкенда GsGit (это реально существует на `api.gsgit.org`)

Базовый URL: `https://api.gsgit.org`
Content-Type для POST: `application/json`. Все ответы — JSON.

### 3.1 `GET /admin/stats` — сводка для дашборда
Заголовки: `X-Admin-Key`.
Ответ `200`:
```json
{
  "logins": 12,
  "devices": 18,
  "quietEnabled": 4,
  "heldPushes": 3,
  "maintenance": "off",
  "latestVersion": "1.0.84",
  "minVersion": "1.0.0"
}
```
Используй как первичную проверку ключа и как источник цифр дашборда.

### 3.2 `GET /admin/devices` — список устройств
Заголовки: `X-Admin-Key`.
Ответ `200`:
```json
{
  "logins": 2,
  "devices": [
    {
      "login": "octocat",
      "count": 1,
      "devices": [
        {
          "name": "HONOR BVL-N49",
          "tzOffsetMin": 180,
          "quietHours": { "start": 23, "end": 8 },
          "heldCount": 0,
          "tokenTail": "a1b2c3"
        }
      ]
    }
  ]
}
```
`quietHours` может быть `null` (тихие часы выключены). `tokenTail` — последние 6 символов
FCM-токена (для различения устройств, не секрет). Отрисуй как раскрывающийся список:
логин → его устройства.

### 3.3 `GET /appconfig` — текущий конфиг приложения (публичный, ключ НЕ нужен)
Ответ `200`:
```json
{
  "maintenanceSoon": "",
  "maintenance": "",
  "latestVersion": "1.0.84",
  "minVersion": "1.0.0",
  "changelog": "+ added ...\n* fixed ...",
  "downloadUrl": "https://github.com/lkolholk-ctrl/gsgit-releases/releases/latest"
}
```
Загружай при открытии редактора конфига, чтобы показать текущие значения.

### 3.4 `POST /admin/appconfig` — изменить конфиг
Заголовки: `X-Admin-Key`. Тело — любые из полей (передавай все, что редактируешь; строки):
```json
{
  "maintenanceSoon": "Maintenance in ~1 hour",
  "maintenance": "",
  "latestVersion": "1.0.85",
  "minVersion": "1.0.80",
  "changelog": "+ ...\n* ...",
  "downloadUrl": "https://github.com/lkolholk-ctrl/gsgit-releases/releases/latest"
}
```
Ответ `200` — обновлённый конфиг целиком (тот же формат, что `GET /appconfig`).
Семантика полей (объясни в UI подсказками):
- `maintenanceSoon` — текст жёлтой плашки «скоро техработы» (пусто = выключено);
- `maintenance` — текст **полной блокировки** всех клиентов (пусто = выключено). **Это
  kill-switch.** Клиенты перепроверяют раз в минуту и сами разблокируются, когда очистишь.
- `latestVersion` — мягкое обновление (диалог «доступно обновление»);
- `minVersion` — принудительное обновление (клиенты старее — не пускаются в приложение);
- `changelog` — текст «что нового»,многострочный (`\n`);
- `downloadUrl` — куда вести за APK.

### 3.5 `POST /announce` — пуш-анонс всем устройствам
Заголовки: `X-Admin-Key`. Тело:
```json
{ "title": "GsGit 1.0.85 is out", "body": "Update from the site.", "url": "https://gsgit.org" }
```
`title` и `body` обязательны, `url` опционален. Ответ `200`:
```json
{ "ok": true, "delivered": 17 }
```
Покажи пользователю «Доставлено на N устройств».

### 3.6 Коды ошибок (обрабатывай ВСЕ)
- `401 {"error":"bad admin key"}` — ключ неверный → на экран входа, «Wrong key».
- `400 {"error":"bad json"}` / `{"error":"title and body required"}` — валидация.
- Сетевая ошибка / таймаут → показать «Server unreachable», не падать.
- Никогда не показывай сырой стектрейс; всегда человекочитаемое сообщение + возможность повторить.

## 4. Экраны приложения (сделай ВСЕ)

Ориентир по UX — существующая панель GlassFiles (дашборд + карточки + список устройств +
кнопки действий). Структура:

### 4.1 Экран входа (Lock screen)
- Поле «X-Admin-Key» (тип password/маскированное), кнопка **Unlock**.
- По Unlock — пробный `GET /admin/stats`. Успех → в панель. Ошибка → сообщение.
- Ничего кроме поля ключа тут не показывать.

### 4.2 Дашборд
Карточки-плитки (данные из `GET /admin/stats`):
- **Devices** (`devices`), **Logins** (`logins`), **Quiet hours on** (`quietEnabled`),
  **Held pushes** (`heldPushes`).
- **Kill-Switch** — крупный статус ON/OFF (из `maintenance`), красный/зелёный. Тап →
  быстрый тумблер: включить = открыть диалог с полем текста maintenance и записать;
  выключить = отправить `maintenance:""`.
- Строка версий: `latest` / `min`.
- Кнопка Refresh (перезапрашивает stats + devices).

### 4.3 App Config (редактор гейтов)
- Форма со всеми полями из 3.3/3.4: maintenanceSoon, maintenance, latestVersion,
  minVersion, changelog (многострочное), downloadUrl.
- При открытии — подтягивает `GET /appconfig` и заполняет поля.
- Кнопка **Save** → `POST /admin/appconfig` → тост «Saved, clients pick up within a minute».
- Подсказки под полями (семантика из 3.4). Валидация версий (формат `x.y.z`).

### 4.4 Announce (композер анонсов)
- Поля Title, Text (многострочное), URL (опц.). Кнопка **Send**.
- `POST /announce` → «Delivered to N devices». После успеха очистить поля.
- Подтверждение перед отправкой («Отправить N устройствам?») — это рассылка всем.

### 4.5 Devices (список устройств)
- Из `GET /admin/devices`: список логинов, разворачивается в устройства (name, tz,
  quiet hours, heldCount, tokenTail).
- Поиск/фильтр по логину и имени устройства.
- Счётчик сверху (logins / devices).

## 5. Требования к качеству (анти-халява — читай обязательно)

GPT, это критично. Не делай «на отвали»:

1. **Обработай КАЖДЫЙ код ответа** из раздела 3.6. Ни один запрос без try/catch и без
   обработки не-200. Сетевые ошибки не должны ронять приложение.
2. **Никаких заглушек данных.** Все цифры и списки — из реальных ответов API, не хардкодь
   примеры из этого ТЗ в UI.
3. **Ключ никогда не в коде и не в логах.** Только localStorage/защищённое хранилище.
4. **Kill-switch и Announce — с подтверждением.** Это действия, затрагивающие всех.
5. **Loading / empty / error состояния** на каждом экране (спиннер при запросе, «нет
   устройств», «ошибка, повторить»).
6. **Идемпотентность форм:** повторный Save/Send не должен дублировать; блокируй кнопку
   на время запроса.
7. **Адаптивность:** телефон и десктоп (панель личная, но открываться может с обоих).
8. **Один самодостаточный деплой:** собери так, чтобы можно было залить как статику
   (или один APK). Никаких внешних сервисов, кроме `api.gsgit.org`.
9. **Комментируй, где `// TODO`,** если что-то не смог доделать — но не пропускай молча.
