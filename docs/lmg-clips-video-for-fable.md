# LMG — клипы (музыкальные видео ICM) для Fable

ICM добавили клипы (Apple Music музыкальные видео, 720/1080/1440p, подписанный стрим — как аудио).
Раздел **Video** в навигации (см. бриф по альбомной). Всё идёт через наш прокси — секрет на устройстве не нужен.

## Доступ и требования

- **Только линкованный ICM-юзер с активной подпиской.** Иначе `403 subscription_required` → показать гейт
  «нужна подписка» (как на premium-аудио).
- Нужен **scope `clips`** в сессии. Он включается **в партнёрской админке ICM по запросу** (byicloud) — это
  делает Стас на стороне ICM; после включения новые сессии (`/session/issue`) получат scope `clips`.
- Все запросы — с `Authorization: Bearer <partner_session_token>` и `X-Partner-User-Id` (прокси форвардит,
  ключ/origin/UA ставит сервер).

## Эндпоинты (через наш прокси — префикс `/lmg/icm`)

Базовый URL: `https://api.gsgit.org/lmg/icm`

- `GET  /clips/search?q=...` — поиск клипов. Ответ: список (id, title, artist, обложка, длительность…).
- `POST /clips/resolve` — запустить подготовку стрима для выбранного клипа (тело — id клипа/трека).
  Ответ: `{ job_id }` (async-пайплайн, как транскод).
- `GET  /clips/job/{job_id}` — статус подготовки. Поллить до готовности → в ответе `file_id` (+ status:
  pending/ready/failed).
- `GET  /clips/stream/{file_id}` — подписанный стрим (720/1080/1440p). Либо URL в JSON, либо 302 на CDN —
  прокси пробрасывает и то, и другое (`Location` форвардится). ExoPlayer играет напрямую.

## Флоу воспроизведения клипа

```
search  ──►  юзер тапнул клип
   │
resolve (POST) ──► job_id
   │
poll GET /clips/job/{job_id}  (backoff ~1-2с, пока status != ready; показать «готовим видео…»)
   │  ready → file_id
GET /clips/stream/{file_id}  ──► подписанный URL (или 302)
   │
ExoPlayer (video surface) играет URL напрямую (байты видео — с CDN ICM, мимо нашего сервера)
```

## Клиент (файлы/слой)

- `IcmApi.kt` — добавить методы: `clipsSearch(q)`, `clipsResolve(id)`, `clipsJob(jobId)`, `clipsStream(fileId)`.
  Пути относительные (`/clips/...`) — база уже `…/lmg/icm`, ничего кроме путей не менять.
- Модели: `IcmClip`, `IcmClipJob { job_id, status, file_id }`, `IcmClipStream { url, quality }` — сверить
  реальные поля по ответам (как делали с `IcmSessionResponse`).
- **ExoPlayer**: тот же плеер, что аудио, умеет видео из коробки — нужна `PlayerView`/video surface на экране
  клипа. Выбор качества 720/1080/1440 (если `resolve`/`stream` принимает параметр качества — прокинуть).
- Экран **Video**: сетка превью (в ландшафте — крупное видео слева, похожие/список справа, см. landscape-бриф).
- Обработать `403 subscription_required` (гейт подписки) и `failed`-джобу (ретрай/ошибка).

## Definition of Done

- Раздел Video: поиск клипов → тап → «готовим…» (job poll) → видео играет (ExoPlayer, напрямую с CDN).
- Гейт подписки на 403. Качество переключается (если API даёт). В ландшафте — раскладка из landscape-брифа.
- Секрета/ключа в APK нет (всё через `/lmg/icm`). Сборка зелёная.

## Что на стороне сервера (готово)

- Прокси уже пропускает `/clips/*` с Bearer+partner-user (v1.0.7), форвардит `Location` для 302-стрима,
  латентность клипов видна в админке отдельной группой `clips`. Правок не требуется.

## Что нужно от Стаса (ICM-сторона)

- Включить **scope `clips`** для партнёрского ключа в админке byicloud (по запросу). Без него — 403/нет scope.
- После включения — проверить, что новая сессия содержит `clips` в `scopes` (видно в ответе `/session/issue`).
