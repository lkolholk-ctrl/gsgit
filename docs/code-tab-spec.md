# Code tab — спецификация и состояние реализации

> Статус на 2026-07-16: стадии 0–6 и дорожная карта до GitHub Notifications 1.0.59 реализованы. Версия
> 1.0.50 добавила типизированное рабочее дерево `A/M/D/R`, 1.0.52 — постоянные вкладки и Quick
> Open, 1.0.53 — Smart Editor, 1.0.54 — безопасный rebase draft, ветки, PR и blame/history,
> 1.0.55 — единый state-holder, атомарное состояние с recovery и ограниченный offline-кеш,
> 1.0.56 — безопасные preview и защита редактора от больших/бинарных файлов, 1.0.57 —
> маршрутизацию поддерживаемых ссылок, а 1.0.58 — реальные GitHub Apps API-данные без ложного
> статуса установки. Версия 1.0.59 завершает Notifications REST inbox.

## Концепт

Нижний таб **Code** = мини-IDE внутри репозитория:
**переосмысленный браузер «рабочего дерева» → существующий редактор (`CodeEditorScreen`) → батч-коммит.**

Это отдельная секция, не равная read-only лендингу репозитория (файловому дереву на входе).
`selectedSection = Code` при заходе в таб.

## Архитектура

- Редактор остаётся **«тупым»**: на вход `content + path + language`, на сохранение — **колбэк save-to-draft**. Редактор НЕ знает про GitHub.
- Всю GitHub-логику (фетч/коммит) владеет **таб Code**, не редактор.
- Внутренняя навигация Code-таба: `browser → editor → back в browser` (свой бэк-стек, не выход из репо).
- **Стрелка назад из корня браузера → выход из Code на лендинг Repo.**
- Детект языка для редактора — по расширению файла.

## Данные: черновик + батч-коммит

### Черновик (draft)
- Незакоммиченные правки = **черновик**, скоуп **`(репо, ветка)`**.
- Персистится **на диск** (переживает смерть процесса).
- **Авто-сохраняется молча** на: выходе из Code / смене ветки / уходе в фон. Без диалогов.
- Авто-восстанавливается при возврате в Code: грязные маркеры на месте, панель «изменения» показывает «N несохранённых».
- Отдельного UI у черновика нет — он виден через панель «изменения», там же и discard.
- Буфер: `path → CodeChange` (`CodeAdd`, `CodeModify`, `CodeDelete`, `CodeRename`).
- Старый дисковый формат `path → content` автоматически мигрирует в `CodeModify`.

### Смена ветки
- Авто-дрфт текущей ветки + загрузка черновика целевой ветки (если есть).
- Дилемма «оставить/сбросить» растворяется — каждая ветка держит свой черновик. Discard остаётся опцией.

### Коммит (Git Data API, батчем)
Один коммит на много файлов:
1. Получить ref (текущая ветка) → sha последнего коммита.
2. Получить tree sha этого коммита (base tree).
3. Для каждого изменённого файла: создать blob → sha.
4. Создать tree (`base_tree` + записи: модификации/добавления через blob sha, удаления через `sha:null`, переименование = удаление старого + добавление нового).
5. Создать commit (message, tree, parent = последний коммит).
6. Обновить ref → новый sha коммита.

Коммитим в **текущую выбранную ветку** (селектор сверху).

### Конфликт (ref уехал во время коммита)
Перефетч ref, предупредить пользователя, дать ретрай. **Без авто-мёрджа.**

## Переосмысленный браузер (changes-oriented)

Отличие от лендинга: лендинг = «читаю репо», Code-браузер = «рабочее дерево с правками».

- **Панель «изменения»**: файлы с незакоммиченными правками + кнопка **«Закоммитить N файлов»** → шит сообщения → батч. (По сути лёгкий Source Control как в VS Code.)
- **Маркеры грязных**: точка у изменённого файла; роллап-точка на папке, где внутри есть правки.
- **«Недавние / открытые»** строкой сверху — быстрый возврат к файлу.
- **Крошки пути + быстрый «вверх»**.
- **Create / rename / move / duplicate / delete** работают как локальные операции черновика.
- Пустая папка представляется файлом `.gitkeep`, потому что Git не хранит пустые директории.
- Фильтр файлов.

## Зафиксированные решения

| # | Решение |
|---|---------|
| 1 | Назад из корня браузера → выход из Code на лендинг Repo. |
| 2 | Смена ветки с правками → авто-черновик per-ветка (без блокировки). |
| 3 | Долговечность правок → черновик персистится на диск. |
| 4 | Create/rename/move/duplicate/delete → реализовано в 1.0.50 как операции черновика. |
| 5 | С 1.0.52 открытые вкладки сохраняются отдельно для каждой ветки. |
| 6 | Конфликт ref → перефетч + предупреждение + ретрай, без авто-мёрджа. |

## Стадии сборки

Дисциплина: **build + commit + device-test на каждой стадии. Без чейнинга.**

| Стадия | Содержание |
|--------|------------|
| 0 | Оболочка Code + внутренний бэк-стек + проводка `selectedSection` (заглушка-экран). |
| 1 | Переосмысленный браузер **read-only** — фетч дерева, крошки, навигация, иконки по типу. |
| 2 | Открыть файл → существующий редактор **read-only** — фетч контента, детект языка, показ. *(Тут же ложится отложенный Phase-2 state-lift редактора — оборачиваем, НЕ переписываем.)* |
| 3 | Правка → **буфер черновика** + маркеры грязных + discard. Ничего не пушится. |
| 4 | **Батч-коммит** via Git Data + шит сообщения. **Единственная реальная рисковая часть (дата-слой), изолирована тут.** |
| 5 | Панель «изменения» + полиш — отдельная поверхность изменений, роллап-маркеры, недавние/открытые. |
| 6 | Прочность — гард смены ветки, персист черновика на диск, обработка конфликтов. |

Стадии 0–6 завершены. Батч-коммит поддерживает add/modify/delete/rename одним tree и обновляет ref
с `force=false`; при гонке ветки UI сохраняет черновик и предлагает retry.

## Workspace 1.0.52

- Открытые вкладки, активный файл и история переходов хранятся на диске в скоупе
  `(репозиторий, ветка)`; после перезапуска возвращается то же рабочее пространство.
- Закрытие вкладки не удаляет draft. Переключение вкладки, Quick Open и переход по истории сначала
  передают последние правки в типизированный draft.
- Quick Open строит индекс по рекурсивному Git tree выбранной ветки, накладывает локальные
  `A/M/D/R` и исключает бинарные файлы из текстового редактора.
- Search repo использует GitHub Code Search для индексированного содержимого и объединяет результат
  с поиском по незакоммиченным локальным draft-файлам. GitHub индекс ищет default branch; это явно
  отмечается в интерфейсе, если workspace открыт на другой ветке.
- Back/forward история ограничена 40 переходами, вкладки — 12 файлами, чтобы состояние оставалось
  компактным и предсказуемым на телефоне.

## Smart Editor 1.0.53

- Автопары для `()`, `[]`, `{}`, кавычек, backtick и `< >` в markup: оборачивание выделения,
  пропуск уже существующего закрывающего символа и парное удаление Backspace.
- Умный Enter учитывает выбранные в настройках tabs/spaces, продолжает текущий отступ, открывает
  вложенный блок и корректно раскладывает курсор между парой скобок. Закрывающая скобка на пустой
  строке автоматически уменьшает отступ.
- Toggle comment поддерживает line-comments для Kotlin/Java/JS/TS/Dart/C/C++/Python/Shell/YAML/
  SQL и block-comments для CSS/XML/HTML/SVG/Markdown, сохраняя логическое выделение.
- Formatter: строгий JSON pretty-print с сообщением ошибки, структурный brace-formatter, безопасный
  line-based markup indent и удаление trailing whitespace для остальных форматов. Скобки внутри
  строк, template/triple-quoted literals и комментариев не влияют на структуру.
- Локальные diagnostics после debounce: JSON parse errors, несбалансированные скобки, XML/HTML
  tags (включая многострочные), YAML tabs, merge markers и trailing whitespace. Панель problems
  ведёт к точной строке; анализ выполняется вне UI thread и ограничен файлами до 1 МБ.
- Полная accessory-панель теперь доступна и в lite-редакторе Code workspace. Status bar показывает
  реальные Tabs/Spaces из настроек и прокручивается по горизонтали на узких экранах.

## Git Workflow 1.0.54

- Для каждого текстового draft отдельно хранится удалённый ancestor. Если ref ветки успел уехать
  во время батч-коммита, draft перебазируется на новый HEAD консервативным line-based diff3:
  независимые изменения объединяются, пересекающиеся сохраняются как `LOCAL / BASE / REMOTE`.
- Conflict Resolver понимает полноценные трёхсторонние маркеры и показывает ancestor отдельно.
  Коммит блокируется, пока в draft остаются неразрешённые conflict markers.
- `branch` в Source Control создаёт серверную ветку от текущего HEAD, атомарно переносит в её
  локальный слот весь `A/M/D/R` draft вместе с ancestors и переключает Code workspace на неё.
- При коммите не в default branch можно сразу создать pull request; заголовок берётся из commit
  message, base — default branch, head — активная ветка.
- В lite-редакторе появились `B`/`H`: настоящий GraphQL blame по диапазонам строк и REST history
  файла. Переход в commit diff и возврат сохраняют вкладки и незакоммиченный draft редактора.

## Reliability 1.0.55

- Один `CodeWorkspaceStateHolder` владеет всем изменяемым состоянием Code workspace: типизированным
  draft, ancestor-базами, вкладками и back/forward-историей. Записи сериализованы, а ревизии не
  позволяют запоздавшей coroutine перезаписать более новое состояние той же ветки.
- Для каждой пары `(репозиторий, ветка)` сохраняется единый атомарный JSON snapshot с версией и
  SHA-256 checksum. Перед заменой основной копии сохраняется последняя проверенная recovery-копия;
  повреждённое или оборванное состояние восстанавливается автоматически.
- Первый запуск 1.0.55 мигрирует существующие draft, ancestor-базы и workspace из прежних
  SharedPreferences без потери незакоммиченных изменений.
- Просмотренные текстовые файлы и директории имеют отдельный best-effort disk cache: записи
  атомарные, одна запись ограничена 2.5 МБ, весь кеш — 24 МБ с вытеснением старых файлов.
  Сеть всегда приоритетна; при транспортной ошибке используется последняя кешированная копия.
- Browser и lite-editor явно отмечают работу из offline cache. Draft при этом продолжает
  сохраняться локально; операции, требующие проверки существования пути, блокируются, если нет ни
  ответа GitHub, ни кешированной директории. Настоящий `404` удаляет устаревшую кеш-копию.
- Offline-кеш можно безопасно очистить средствами Android: он не содержит единственную копию draft
  и не участвует в atomic recovery рабочего пространства.

## Preview 1.0.56

- Классификация файла централизована и одинакова для Browser, Quick Open и Editor: text, Markdown,
  JSON, SVG, raster image и unsupported binary. Quick Open показывает previewable-файлы, а архивы,
  базы, шрифты, исполняемые и прочие бинарные форматы не отправляет в текстовый редактор.
- Markdown Preview использует GitHub-aware renderer с относительными ссылками и изображениями;
  переключение Edit/Preview не создаёт отдельную копию draft.
- JSON Preview валидирует корень object/array, строит безопасное форматированное представление с
  syntax highlighting и показывает ошибку вместе с исходным текстом, не изменяя содержимое файла.
- SVG Preview рендерится из текущего текста редактора, поэтому незакоммиченные изменения видны сразу.
  PNG/JPEG/GIF/WebP/BMP/ICO загружаются через авторизованный Git Blobs API и открываются в canvas с
  pan/zoom; это работает и для приватного репозитория без передачи токена в URL.
- До Base64 decode проверяются серверный `size`, ожидаемый encoded-size и фактический массив bytes.
  Текст ограничен 1 МБ, изображения — 8 МБ. Неизвестное расширение дополнительно проходит проверку
  NUL/control bytes и строгий UTF-8 decoder.
- Большой, бинарный или не-UTF-8 файл не попадает в Compose editor. Известные из tree тип/размер
  блокируются до fetch; проблема, найденная при загрузке, открывает guard-экран с причиной,
  типом/размером и переходом на GitHub. Старый offline cache проходит те же лимиты и
  content-sniffing, поэтому не может обойти защиту.

## Supported links и GitHub Apps 1.0.57

- `github.com` и `www.github.com` маршрутизируются одинаково. Поддерживаемые repository URL
  открываются внутри GsGit, а системные страницы GitHub, загрузки и неизвестные маршруты безопасно
  передаются внешнему браузеру без повторного перехвата самим приложением.
- Экран Apps показывает реальный Android-статус supported links и ведёт прямо в системный раздел
  «Открывать по умолчанию». Автоматическая верификация `github.com` невозможна без файла
  `assetlinks.json` на стороне GitHub, поэтому приложение не показывает ложный verified-статус.
- Добавлен навигационный URI `gsgit://integrations/github-app`. Он только открывает экран Apps и
  намеренно не принимает решения об авторизации: параметры custom scheme могут быть подделаны.
- Codex и Claude Code устанавливаются с официальных страниц GitHub Apps. Выбор `All repositories`
  или `Only select repositories`, аккаунта/организации и подтверждение разрешений выполняет GitHub.
- `/user/installations` остаётся дополнительной проверкой для совместимого GitHub App user token.
  Обычный PAT не может читать установки сторонних Apps, поэтому его `403` больше не блокирует
  onboarding и объясняется в интерфейсе вместо ложного статуса подключения.
- Приватный ключ GitHub App и installation token не создаются на Android-устройстве. Завершающий
  callback принадлежит провайдеру App; после возврата GsGit обновляет доступный ему статус.

## GitHub Apps API Truth 1.0.58

- Карточки Codex и Claude получают живые имя, owner, App ID, описание, permissions, events и даты
  из `GET /apps/{app_slug}`. Этот ответ подтверждает идентичность и возможности App, но не установку.
- `GET /user/installations` отображается отдельным блоком вместе с HTTP-кодом. Его результат
  относится только к App, выдавшему текущий GitHub App user access token; `403` PAT не превращается
  ни в «не установлено», ни в «установлено».
- Для любого доступного токену репозитория собираются реальные Apps из check-runs последних пяти
  коммитов, список workflow и все `uses:` из читаемых YAML, а также имена Claude/OpenAI-related
  Actions Secrets. Значения Secrets GitHub API никогда не возвращает.
- Для Checks, Workflows и Secrets интерфейс показывает отдельные HTTP-коды и ошибки полномочий.
  Пустая выборка всегда подписана как отсутствие наблюдаемого evidence, а не отсутствие установки.
- Если доступен совместимый App user token, сохраняется полный installation flow: selection
  `all/selected`, permissions, events, timestamps, suspension и управление выбранными репозиториями.

## GitHub Notifications 1.0.59

- Inbox использует глобальный `GET /notifications` и repository endpoint с реальными параметрами
  `all`, `participating`, `since`, `before`, `page` и максимальным `per_page=50`. Пагинация берётся
  из GitHub `Link`, а пустой ответ больше не скрывает HTTP-ошибку авторизации.
- Интерфейс показывает фактические subject types и reasons из ответа GitHub, поиск по загруженным
  тредам, repository scope, временные окна, unread/loaded counters, `X-Poll-Interval`,
  `Last-Modified`, HTTP status и текущий API rate limit.
- Полная карточка треда загружается через `GET /notifications/threads/{id}` и показывает subject,
  latest comment URL, timestamps, reason, read-state и все исходные API URL.
- Поддержаны все mutation endpoints уведомлений: mark thread read, mark thread done, mark all read,
  mark repository read, subscribe/ignore/delete thread subscription.
- В той же карточке читается и меняется repository subscription: watch, ignore repository и
  unwatch. `404` отсутствующей явной подписки отображается как состояние, а не как фиктивная ошибка.
- Background worker соблюдает серверный `X-Poll-Interval`, различает transport/server и auth
  failures, дедуплицирует уже показанный набор и выводит Android InboxStyle до пяти свежих тредов.
- GitHub ограничивает Notifications REST endpoints classic PAT с `notifications` либо `repo` scope;
  fine-grained PAT и GitHub App tokens не поддерживаются самим GitHub, что явно объясняется при 403.
