// why gsgit
Three things other clients don't do
⌘

Sign in without tokens

GitHub App device flow: no personal access tokens to create, paste or leak. A short-lived session lives in hardware-encrypted storage, refreshes itself, and can be revoked from GitHub at any moment. New device on your account? Every other device gets a security alert.

⚡

Instant push notifications

An open-source notification backend turns GitHub webhooks into pushes that reach your phone in ~2 seconds — no battery-draining polling. Rich expandable texts and ten separate Android channels: mute stars, keep CI, or the other way round.

▶

CI that feels native

Workflow runs live-refresh on screen while they execute. Artifacts download in the background with progress and Cancel, land straight in system Downloads, unzip themselves. Dispatch and re-run workflows right from the phone.

// how pushes work
No polling. No middlemen you can't read.
GitHub webhook → api.gsgit.org → FCM → your phone (~2 s)

The server stores exactly two things: your GitHub login and the device push token. Nothing else. The backend is open source in the same repository — run your own with Docker Compose if you like.

// security & privacy
Built like you'd build it yourself
  • Open source — the app, the push backend, this page.
  • Tokens never leave the device — sessions sit in encrypted storage; there is no PAT anywhere in the product.
  • Revoke anytime — one click in GitHub settings cuts all access instantly.
  • No analytics, no ads, no tracking — read the privacy policy, it's refreshingly short.
