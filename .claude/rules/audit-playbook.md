# Playbook: аудит экранов на баги и недочёты

Запускать **по одному экрану за сессию** (`/clear` между ними). На каждый экран Claude Code
СНАЧАЛА выдаёт отчёт находок, и только после подтверждения чинит. Молча не править.

## Процесс на один экран

1. Прочитать файл экрана + связанные под-Composable'ы и его вызовы `GitHubManager.*`.
2. Пройти по чек-листу ниже.
3. Выдать отчёт в формате:
   ```
   ЭКРАН: GitHubXxxModule.kt
   [CRASH]  <что, строка, почему падает, как чинить>
   [BUG]    <неверное поведение, строка, фикс>
   [SMELL]  <недочёт/риск, строка, рекомендация>
   COVERAGE: какие состояния (loading/empty/error) не обработаны
   ```
4. После ревью — чинить по одному пункту, билд после каждого, поведение не ломать.

## Чек-лист (Compose / Android specifics)

**Падения (CRASH):**
- `LazyColumn`/`LazyRow` без стабильного `key` или с дублирующимися ключами (в проекте уже ловили duplicate-key — проверять везде).
- Небезопасный парсинг JSON: `getJSONObject`/`getString` вместо `opt*`, `!!`, индексы без проверки длины.
- `IndexOutOfBounds`/`NPE` на пустых/частичных ответах API.
- Работа с `ApiResult`, где `success == false` не обрабатывается и код идёт дальше как при успехе.

**Стейт и рекомпозиция (BUG/SMELL):**
- `mutableStateOf` без `remember` (пересоздаётся каждую рекомпозицию).
- Стейт не переживает поворот/смерть процесса (нужен ViewModel или `rememberSaveable`).
- Нестабильные лямбды/параметры → лишние рекомпозиции; кандидаты на `remember`/`derivedStateOf`.
- Чтение часто меняющегося state на уровне всего экрана → рекомпоз всего вместо куска.

**Side-effects и корутины (BUG):**
- Сетевые вызовы прямо в теле Composable вместо `LaunchedEffect`/VM.
- Неверные ключи `LaunchedEffect` (перезапуск на каждую рекомпозицию или не запускается вовсе).
- Flow собирается через `collectAsState` вместо `collectAsStateWithLifecycle` (работа продолжается вне экрана).
- Нет отмены/таймаутов; долгие операции на main.

**Сеть/ресурсы (BUG):**
- Прямой `openConnection()` в экране (см. протечки в CLAUDE.md) — стрим/коннекшн не закрыт, мимо retry/etag/proxy.
- Пагинация: не использует `parseNextPage`, дубли элементов, не останавливается на последней странице.
- Игнор rate-limit (`isRateLimitLow`) там, где идёт серия запросов.

**UX-недочёты (SMELL):**
- Нет loading/empty/error состояний (пустой экран вместо «ничего не найдено»/«ошибка, повторить»).
- Не показывается ошибка API пользователю.
- Нет `contentDescription` на иконках-кнопках (доступность).

## Definition of Done аудита экрана

Отчёт выдан, CRASH и BUG устранены, добавлены недостающие loading/empty/error,
прямой HTTP убран в ядро, билд зелёный, поведение проверено вручную.
